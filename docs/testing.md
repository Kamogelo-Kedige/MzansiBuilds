# Testing

## Testing Approach
Because of the limited implementation timeline, the testing focus is on core logic and validation rather than full end-to-end browser automation.

## Automated Tests
Automated tests cover:
- input validation
- helper functions
- project-related utility logic

## Manual Testing
Manual testing covers:
- signup and login flow
- project creation
- commenting
- milestone creation
- collaboration request submission
- celebration wall rendering

## Why This Approach Was Used
The selected approach balances speed and quality while still demonstrating structured engineering practice.

## Future Testing Improvements
- end-to-end browser tests
- integration tests against Supabase test environment
- UI component tests
